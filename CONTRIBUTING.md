# Contributing to Governed Run

Thank you for your interest in contributing to Governed Run! We welcome contributions that help shape the future of Continuous Software Generation (CSG) and bounded autonomous agentic systems.

## Getting Started

1. **Fork the Repository**: Create your own copy of this repository on GitHub.
2. **Clone Locally**: Clone your fork to your development environment:
   ```bash
   git clone https://github.com/carlosrivera/governed-run.git
   ```
3. **Install Dependencies**: Install package dependencies using bun or npm:
   ```bash
   bun install
   ```
4. **Run Development Server**: Start the local Astro server:
   ```bash
   bun run dev
   ```

## Development Workflow

- **Branching Policy:** Create feature branches from the `main` branch. Use descriptive names like `feature/dynamic-filtering` or `fix/rss-discoverability`.
- **Code Standards:** Ensure all Astro components and TypeScript files conform to the project formatting and compile checks:
   ```bash
   bun run build
   ```
- **Commit Messages:** Commit changes using clear, concise commit messages. Keep titles under 50 characters and provide descriptive details if necessary.

## Submitting Pull Requests

1. Commit your changes to your feature branch.
2. Push your branch to your GitHub fork.
3. Open a Pull Request (PR) from your feature branch to the `main` branch of this repository.
4. Describe the modifications made, why they are needed, and how you verified them.
5. Participate in the PR review process to help merge the code.
